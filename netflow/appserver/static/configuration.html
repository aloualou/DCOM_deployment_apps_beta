Step 3

Get more value out of NetFlow by enabling additional inline analytics in NetFlow Integrator and enabling the corresponding Splunk App panels below. Place a checkbox for the selected panel and click on the "Save" button below.

Cyber Threat Dashboard Settings



This dashboard shows cyber threat traffic based on Security Rule Set of NetFlow Integrator. Make sure you have the following Rules enabled:
  • 10050: Botnet Command and Control Traffic Monitor
  • 10051: APT1 Traffic Monitor
  • 10052: Host Reputation Monitor

Cisco ASA Monitor Settings



These dashboards provide visibility into your network traffic passing through your Cisco ASA devices. It allows you to view the top bandwidth consumers, violators, connectors, and destinations during the time interval which is configurable through each rule. (e.g. 30 seconds)

These dashboards show traffic based on Cisco ASA NSEL NetFlow. Make sure you have the following Rules enabled:

  • 10018: Top Bandwidth Consumers for Cisco ASA
  • 10019: Top Traffic Destinations for Cisco ASA
  • 10020: Top Policy Violators for Cisco ASA
  • 10021: Top Hosts with most Connections for Cisco ASA

Palo Alto Networks Settings



NetFlow views are based on NetFlow data produced by Palo Alto Networks devices and converted to syslog messages by 3rd party software - NetFlow Integrator.

These dashboards show traffic based on Palo Alto Networks devices. Make sure you have the following Rules enabled:

  • Enable NetFlow Integrator Palo Alto Networks Rules (10030 through 10035) and Converter (20093)

Network Traffic by CBQoS Settings



This dashboard shows traffic by Differentiated Service (DSCP) on your network. Make sure you have the following Rule enabled:
  • 10065: Traffic by CBQoS

Traffic by Subnets Settings



This dashboard reports top bandwidth consumers for each monitored subnet. Make sure you have the following Rule enabled:
  • 10011: Network Subnets Monitor

Traffic by Autonomous Systems Settings



This dashboard reports traffic by all Autonomous Systems. Make sure you have the following Rule enabled:
  • 10066: Traffic by Autonomous Systems

Traffic by Host Pairs Settings



This dashboard shows top traffic conversations between host pairs. Make sure you have the following Rule enabled in Network traffic and device monitoring Rule Set:
  • 10064: Top Host Pairs Monitor

Traffic by Connections Settings



This is a group of dashboards that shows top traffic by the number of connections. Make sure you have the following Rule enabled in Network traffic and device monitoring Rule Set:
  • 10063: Top Connections Monitor

Services Response Time Settings



This dashboard shows an average response time of each monitored service. Make sure you have the following Rule enabled:
  • 10017: Services Performance Monitor

DNS Security Settings



This dashboard shows an average response time and top 10 users of each monitored DNS server. Make sure you have the following Rule enabled:
  • 10004: DNS Monitor

GEO IP Monitor Settings



This dashboard shows geographical locations of monitored hosts, and traffic statistics. Make sure you have the following Rule enabled:
  • 10040: Hosts Geographical Location Monitor

Watched Interfaces



Create watched-interfaces.csv file for the Watched Interfaces Utilization dashboard (see watched-interfaces.csv.sample), where
management_ip      is the management IP address of the device
if_name      is the name of the interface (received from SNMP polling)

Interface Groups



Create interface-groups.csv file for the Interface Groups dashboard. Specify Interface groups to view the aggregated traffic for grouped interfaces (see interface-groups.csv.sample), where
management_ip      is the management IP address of the device
if_name      is the name of the interface (received from SNMP polling)
if_group      is the name of the group where the interface belongs
