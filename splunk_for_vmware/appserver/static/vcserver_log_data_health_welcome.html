The VMware Virtual Center Server Log Data Health view helps administrators verify that their data is looking healthy and usable.
The guidance provided assumes default logging settings for vCenter with regards to rotation and logging level.
Checklist:
  1. Source and sourcetype counts over all time: These values count the total number of log sources and sourcetypes from vCenter Server machines with the VC Add-on installed. Each source represents a distinct log file indexed and each sourcetype represents a distinct type of VC log indexed. The data are independent of the time range selected and are over "All time"
  2. VC Forwarding Status: this table shows each vCenter (vCenter running the Splunk Add-on for vCenter) the index has received data from in the last 30 minutes.
  3. Number of events over time: these are counts of events in the said log files
    • vpxd logs: the time range is fixed at 10min and you should see 500 to 10,000 events per minute per VC
    • cim-diag logs: you should see 100 to 150 events per hour per VC
    • vpxd-profiler logs: you should see 15,000 to 25,000 events per 10 minutes per VC
