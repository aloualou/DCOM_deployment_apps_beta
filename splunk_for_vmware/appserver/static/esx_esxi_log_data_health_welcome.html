The VMware ESX/ESXi Log Data Health page helps administrators assess the health of their infrastructure.

Checklist:
  1. Overview of logs by host and log: Shows all the ESX/i hosts from which log data is being received, the actual log source of the data and number of events per log source. It is good practice to verify that all the expected ESX/i hots and log sources are in the table.
  2. Potential gaps in Retrieved Log Data: Indicates gaps in log data received from ESX/i hosts, by host and log type (sourcetype). Gaps can occur due to timing offsets between log data collection by Splunk for VMWare and log rotation on the ESX/i hosts.
  3. Log event count over time: Shows log events aggregated into 30 minute buckets by log type (sourcetype)
